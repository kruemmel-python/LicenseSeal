07 IDE/LSP and Inbound Paste Protection

Objective

LicenseSeal helps identify missing markers early in development environments and flag risky code pastes.

Start the LSP:

Install the LSP Extension:

python -m pip install -e ".[lsp]"

Start the server:

licenseseal lsp

The LSP server is integrated in settings depending on the editor.

One-time check without editor

licenseseal lsp-check path\to\file.py

License validation:

licenseseal lsp-validate .

License Action

If a file lacks a LicenseSeal marker, the LSP can offer a quick-fix action:

LicenseSeal Marker injizieren

Advantage: Developers don’t need to leave the IDE.

Inbound Paste Protection

Inbound Paste Protection detects large pasted code blocks and checks them for:

  • License Seal Boundary Marker
  • foreign watermarks
  • Copyright Indicators
  • structural similarity to known components
  • Risky license warnings

Possible warning:

Warnung: Eingefügter Code weist starke Ähnlichkeit zu einer inkompatiblen Komponente auf.

Recommended Policy

For Companies:

RiskRecommendation
AGPL/GPL in proprietary projectError
Unknown large pasteWarning
License seal from an external organizationError
License Warning (MIT/BSD-compatible)Info/Warning

Limitations

The LSP cannot prove every license violation. It is a shift-left warning system, not a legal decision-maker.

Exercise

  • Start the LSP
  • Open an unmarked file
  • Check if the diagnosis appears
  • Use the Quick Fix action
  • Add a large foreign code block to a test file and observe warnings
