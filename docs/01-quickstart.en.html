01 Quick Start: Protect Your First Project

Goal of this chapter

You protect a small example project, check the result, and understand the workflow.

Create an Example Project

Create a folder:

mkdir C:\tmp\hello-seal
cd C:\tmp\hello-seal

Create a Python file:

@"
def add(a, b):
    return a + b

print(add(2, 3))
"@ | Out-File -Encoding utf8 app.py

On macOS/Linux:

mkdir -p /tmp/hello-seal
cd /tmp/hello-seal
cat > app.py <<'PY'
def add(a, b):
    return a + b

print(add(2, 3))
PY

Select project:

licenseseal inject . --license MIT --owner "Example Corp" --project "hello-seal"

Expect:

LicenseSeal result
==================
injected: 1

View file

Open app.py. A LicenseSeal block should be inserted at the beginning.

The actual code remains executable:

python app.py

Expectation:

5

Review Project

licenseseal audit .

Expectation:

Files missing boundary: 0

Use Dry Run

A dry run shows what would happen without changing files:

licenseseal inject . --license MIT --owner "Example Corp" --dry-run

Update the marker

If you want to update existing markers:

licenseseal inject . --license MIT --owner "Example Corp" --update

Remove the marker

Only for testing or intentional removal:

licenseseal remove .

Then:

licenseseal audit .

Now the file should appear unmarked.

Safe working with backups

licenseseal inject . --license MIT --owner "Example Corp" --backup

LicenseSeal then creates backup copies before modifying files.

Does LicenseSeal change my program logic?

Does LicenseSeal change my program logic?

Normal inject command adds comment blocks. For typical source code files, this does not change the program logic.

Do I need to mark each file individually?

No. You can specify the project folder. LicenseSeal automatically detects supported file types.

What should I do with generated files?

Generated files should mostly be excluded:

licenseseal inject . --exclude-dir generated --exclude-dir dist

Exercise

  • Create a sample project with two files
  • Run licenseseal inject .
  • Run licenseseal audit .
  • Remove the markers with licenseseal remove .
  • Re-run audit and compare the results
