12 Supply Chain Attacks and License Conflicts

Objective

LicenseSeal should not only mark tokens but also prevent injection of incorrect or conflicting license information.

What is SCA?

SCA stands for Software Composition Analysis. It involves reviewing project metadata and dependencies to identify potential license risks.

Supported Manifest Files

License Seal can evaluate, among other things:

  • pyproject.toml
  • package.json package.json
  • pom.xml pom.xml
  • Cargo.toml Cargo.toml
  • go.mod go.mod
  • requirements.txt requirements.txt

Review Project

licenseseal sca check . --license MIT

With error code during conflicts:

licenseseal sca check . --license AGPL-3.0 --fail-on-error

Inject with SCA

By default, the following checks are performed during Inject:

licenseseal inject . --license MIT --owner "ACME GmbH"

When a conflict is detected, LicenseSeal can warn or block

Force and Skip

Use only when necessary:

licenseseal inject . --license AGPL-3.0 --owner "ACME GmbH" --force

Skip SCA completely:

licenseseal inject . --license MIT --owner "ACME GmbH" --skip-sca

Recommended Workflow

1. sca check ausführen
2. Konflikte prüfen
3. Lizenzentscheidung dokumentieren
4. inject ausführen
5. audit in CI aktivieren

Exercise

  • Create a package.json with "license": "MIT"
  • Run the command licenseseal sca check . --license MIT
  • Run with --license AGPL-3.0
  • Compare warnings and exit codes
