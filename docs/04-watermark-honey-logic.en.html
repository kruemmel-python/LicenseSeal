Chapter 4: Watermarking and Honey-Logic

Objective

This chapter explains additional recognition signals beyond standard license markers.

Types of Watermarks

LicenseSeal supports multiple signal types:

TypeDescription
StrengthBoundary Marker
visible comment blockstrong on direct copy
Zero-Width Watermarkinvisible Unicode characters
resistant to header removalSemantic Watermark
small semantic structural featuresrobust against formatting
Honey-Dotsharmless mathematical code sentinels
strong statistical indicatorMulti-Language Honey-Logic
Sentinels for Python, JS/TS, Go, Rust, JavaPolyglot Projects

Embed watermark

Use the help to see the exact options of your build:

licenseseal watermark --help
licenseseal watermark embed --help

Typical workflow:

licenseseal watermark embed src/app.py --message "ACME:payment-service"

Extract watermark:

licenseseal watermark extract src/app.py

Honey-Dots

Honey-Logic is executable, harmless logic with rare constants and operator sequences. It is not intended to sabotage behavior or influence other systems. It only serves for later recognition of one's own software components.

Create Multi-Language Honey-Logic:

licenseseal honey-multilang --help

Example idea:

licenseseal honey-multilang --language python --project "payment-service" --owner "ACME GmbH"
licenseseal honey-multilang --language go --project "payment-service" --owner "ACME GmbH"
licenseseal honey-multilang --language typescript --project "payment-service" --owner "ACME GmbH"

Why Honey-Logic works

A single name can be changed. That’s why LicenseSeal evaluates not only names but also:

  • Constants
  • Operator Precedence
  • AST Form
  • Control Flow
  • Bit Shifts and Masks
  • project-dependent seeds

Detection

Honey-Logic is used as an Evidence Signal by Firehose, Interceptor, and Scanners. A hit is not an automatic verdict but a strong technical indicator.

Caution with production code

Only add Honey-Logic where it:

  • does not cause performance issues,
  • does not distort professional logic,
  • has been approved through Code Review,
  • was documented and internally approved has been.

Exercise

  • Create a Honey-Logic snippet for Python
  • Add it to a test project
  • Copy the file into a second folder
  • Use Firehose or Interceptor to identify the hit
