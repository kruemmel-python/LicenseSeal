00 Installation and Basic Concepts

Goal of this chapter

After this chapter, you can install LicenseSeal, verify the CLI works, and understand the key terms.

Prerequisites

You need:

  • Python 3.10 or newer
  • a terminal, for example PowerShell, Windows Terminal, Terminal.app, or Bash
  • optional Git if you want to use CI/CD, Bot, or Firehose Scans
  • Optional Docker/Redis/PostgreSQL for enterprise scenarios

Check Python:

python --version

Check pip:

python -m pip --version

Install from project folder

Switch to the extracted project folder:

cd C:\LICENSE_SEAL

Install the base version:

python -m pip install -e .

Install the full version with all optional extras:

python -m pip install -e ".[full]"

On macOS/Linux, the command is identical:

python -m pip install -e ".[full]"

Individual Add-ons

If you don’t want to install all dependencies, you can select specific add-ons:

python -m pip install -e ".[crypto]"
python -m pip install -e ".[lsp]"
python -m pip install -e ".[enterprise]"
python -m pip install -e ".[queue]"
python -m pip install -e ".[control-plane]"
python -m pip install -e ".[reports]"

Multiple add-ons can be combined:

python -m pip install -e ".[crypto,lsp,enterprise]"

Check installation

licenseseal --help

Expectation: You see a list of commands like inject, audit, compare, watermark, firehose, sca, graph, intercept, and others.

Important Terms

Marker

A marker is a machine-readable block at the beginning of a file. It describes license, project, owner, and optionally signature information.

Boundary

A clear boundary in the code that indicates: From here on, a LicenseSeal-protected section begins.

Audit

An audit checks whether files are correctly marked and whether signatures remain valid.

A content audit checks whether files are correctly marked and whether signatures remain valid.

A digest is a cryptographic fingerprint of the content. Even minor changes to the relevant content produce a different digest.

A watermark is an additional recognition signal. It can be visible, invisible, or semantic.

A watermark is an additional recognition signal. It can be visible, invisible, or semantic.

Honey-Dots

Harmless, functionally correct code snippets with very specific mathematical structures. They serve as strong indicators if they reappear in external repositories.

Registry

The Registry stores known projects, fingerprints, scan results, and evidence items.

Firehose

The Firehose is the continuous scanner. It checks candidate repositories and reports evidence to the Registry.

Typical installation error: Multiple top-level packages

When you see:

Multiple top-level packages discovered in a flat-layout

Setuptools finds multiple package directories. This project already includes the necessary fix in pyproject.toml:

[tool.setuptools.packages.find]
include = ["licenseseal", "licenseseal.*"]
exclude = ["Notebook_LM", "Notebook_LM.*", "tests", "tests.*"]

First diagnosis:

If a command is not found:

python -m licenseseal --help
