Binary Provenance

Objective

Source markers often disappear during compilation. Binary Provenance embeds provenance metadata into build artifacts or makes it auditable.

Supported Ideas

LicenseSeal provides assistance for:

  • -ldflags -ldflags
  • C/C++ Section Source
  • JAR Manifest Metadata
  • Binary Audit per Byte-Scan

CLI

licenseseal binary --help
licenseseal binary create --help
licenseseal binary append --help
licenseseal binary audit --help

Go Example

Generate Provenance Data:

licenseseal binary create --project "payment-service" --owner "ACME GmbH" --output provenance.json

Build with Go LDFlags:

go build -ldflags "-X main.LicenseSealProvenance=$(Get-Content provenance.json -Raw)" .

In practice, you often minify or escape the JSON string beforehand.

C/C++ Example

LicenseSeal can generate a small C file that embeds provenance data into a section. It is then included in the build.

Example Concept:

__attribute__((section(".note.licenseseal")))
const char LICENSESEAL_PROVENANCE[] = "{...}";

Java Example

For Java artifacts, metadata can be stored in META-INF/MANIFEST.MF

LicenseSeal-Project: payment-service
LicenseSeal-Owner: ACME GmbH
LicenseSeal-Digest: ...

Verify Binary

licenseseal binary audit .\app.exe
licenseseal binary audit .\service.jar

Best Practices

  • Include provenance in the release build
  • Save build logs and commit SHA.
  • Link binary provenance to source audit.
  • Additionally sign release artifacts.
  • Document reproducible build recipes.

Exercise

  • Generate a provenance file
  • Attach it to a test binary
  • Run the licenseseal binary audit command
  • Verify that metadata is recognized
